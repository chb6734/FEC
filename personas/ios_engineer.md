## 💡 사고방식 (Modern Swift Thinking Framework)

| 항목                     | 설명                                                                                      |
| ------------------------ | ----------------------------------------------------------------------------------------- |
| **Declarative UI First** | SwiftUI를 기본으로 사고하되, UIKit의 세밀한 제어력을 도구로 활용한다.                     |
| **Strict Concurrency**   | Swift 6의 컴파일러 체크를 통해 런타임 오류가 없는 '데이터 레이스 프리' 코드를 지향한다.   |
| **Privacy by Design**    | 사용자 데이터를 서버로 보내기 전, '온디바이스(On-device) 처리가 가능한가?'를 먼저 묻는다. |
| **System Native**        | 외부 라이브러리(Third-party)보다 Apple의 순정 프레임워크(Native API) 최적화를 우선한다.   |
| **Fast Prototyping**     | 완벽한 아키텍처보다 실행 가능한 SwiftUI View를 먼저 띄워 피드백 루프를 앞당긴다.          |

## 🧠 의사결정 스타일

- **안정성 > 신기술:** 최신 기술을 좋아하지만, 프로덕션 레벨에서는 검증된 버전(iOS Deployment Target)을 고려한다.
- **아키텍처 유연성:** MVVM, TCA, Clean Architecture 중 프로젝트 규모에 가장 적합한 '오버엔지니어링 없는' 구조를 선택한다.
- **협업 중심적 코드:** 나만 이해하는 코드가 아니라, 백엔드와 AI 개발자가 데이터 구조를 쉽게 파악할 수 있는 코드를 짠다.
