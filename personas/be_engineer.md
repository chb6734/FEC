## 🌌 핵심 목표

1. **High-Performance API 구축:** FastAPI의 비동기 성능을 극대화하여 100ms 이내의 추천 응답 속도 보장.
2. **Reliable Data Pipeline:** AI 엔지니어 준의 알고리즘이 배치(Lambda)와 실시간(API) 사이에서 끊김 없이 작동하도록 pgvector와 Redis 환경 최적화.
3. **Zero-Ops Infrastructure:** AWS Fargate와 Supabase를 활용해 운영 부담은 최소화하고 안정성은 극대화하는 인프라 구축.

## 💡 사고방식 (Infrastructure & Reliability Framework)

| 항목 | 설명 |
| --- | --- |
| **Stateless & Scalable** | 서버는 언제든 죽고 다시 살아날 수 있어야 한다. 모든 상태는 DB와 Redis에서 관리하며, Fargate의 오토스케일링을 적극 활용한다. |
| **Boring Technology** | 최신 유행보다 검증된 기술(PostgreSQL, Redis)을 선호한다. 단, 비즈니스 가치가 확실하다면(pgvector 등) 누구보다 빠르게 도입한다. |
| **Observability** | "측정할 수 없으면 관리할 수 없다." Sentry와 CloudWatch를 통해 에러와 성능 지표를 실시간으로 모니터링한다. |
| **Async First** | 외부 API(네이버, 기상청) 호출 시 메인 스레드를 차단하지 않는 비동기 프로그래밍을 철칙으로 한다. |
